Data Preprocessing

Firstly, the types of all columns in the data set were examined and the object types were converted to categorical types, because the python programming language can perceive the object type as a string. If all values in a feature are different or the same, these features such as type, url, have been removed so that the model can be generalized. Then, missing values were determined by using visualization techniques. There were features with few missing values and one column with more than 50% missing values. It was examined whether the reason for the emptiness was random or not, and when it was decided that it was random, the column was deleted. The KNN imputer which is a machine learning technique, was used to fill in other missing values. Imputation was performed in a uniform way by looking at the 5 closest neighbors. Before using KNN imputer, the data was scaled. The most commonly used method for outlier analysis is the Interquartile range (IQR) calculation. However, a feature-based outlier analysis is made with the created boxplots. With the more advanced Local Outlier Factor technique, outliers are detected within the context. Values that are truly outliers are found. The parameters used for this technique are that n_neighbors is 20 and contamination is 0.1. After that, outliers were deleted from the data set. Since there are more numeric values in the dataset, categorical variables are converted to dummy format. Also duplicate lines were detected and deleted. When the target column is examined, the multinominal label values are unbalanced. For this reason, by using Synthetic Minority Oversampling Technique (SMOTE), all data has been increased by both copying and synthesizing to reach the number of the tag that has maximum number of instances. To use this technique, Label Encoder was used on the target column beforehand.
